## 云端部署就绪化（v3.3）
- 用户要求「部署到云端」，让团队（含异地）能访问带后端的考核平台。
- 重要约束：WorkBuddy 内置 cloudstudio-deploy 仅支持静态站点，跑不了 Node 后端；而本平台是「前端 + Node 后端」一体服务，纯静态托管会导致 API 全部失效、数据各自存浏览器互不相通。因此一键静态部署不可用。
- 已做的云就绪改造：
  - `server.js` 监听改为绑定 `0.0.0.0`（云平台容器外部可访问）。
  - 新增可用环境变量 `DB_PATH`（默认 `./data/db.json`），云部署挂持久盘时设为盘内路径防数据丢失。
  - 新增 `package.json`（start: `node server.js`，engines node>=18，零外部依赖）。
  - 新增 `.gitignore`（忽略 node_modules/data/）。
  - 新增 `README.md`：含 Render 免费部署步骤（连 GitHub → Web Service → start `node server.js` → 挂 Persistent Disk `/data` 并设 `DB_PATH=/data/db.json`）、Railway/Fly 备选、权限与数据说明。
- 冒烟测试（PORT=3210, DB_PATH=测试文件）：GET / 200、公开 /api/config 200、免登录建工单 200、免登录审核 403、管理员审核 200、DB_PATH 落盘生效，全部通过。
- 仍未完成的真实上线：需用户用自己的免费云账号（Render/Railway）连接仓库完成部署一步；本环境无外部账号无法代为推送。已向用户说明并给出精确步骤；另提供「本机+公网隧道」即时方案作为免注册的备选。
