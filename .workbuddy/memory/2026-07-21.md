## 后端化（v3：Node + 前端改 API 驱动）
- 用户要求：(1) 升级单属主 = 被扣分人（按此人汇总扣分），扣分人 = 执行检查/扣分的执行者；(2) 做后端。
- 架构：`server.js`（Node 内置 http/crypto/fs，零外部依赖，JSON 文件存储于 `data/db.json`）+ `index.html`（前端改为 fetch API 驱动，含登录界面）。
- 运行：在目录执行 `node server.js`（默认端口 3000，可用 `PORT=3100 node server.js` 覆盖）。后端启动后浏览器开 `http://localhost:3000`。
- 账号体系（管理员统一创建）：admin / reviewer / user 三种角色，scrypt 密码哈希 + Bearer token 会话。
- 默认管理员：用户名 `admin` 密码 `admin123`（首次启动自动建，需尽快在「系统设置」改密码/建成员）。
- 审核权限服务端强制：仅 role=admin 或 reviewer 才能 PUT `/api/orders/:id/review`；普通成员尝试返回 403（已测）。
- 配置项（错误原因/扣分数）仅 admin 可增删；成员与角色（指定审核人）在「系统设置→成员与审核人」管理。
- 个人扣分汇总按 `升级单属主(被扣分人)` 聚合：最终 = 原始 − 申诉通过减免。
- 统计表支持导出本月 CSV（含工单明细 + 个人扣分汇总，带 BOM 防乱码）。
- 已验证：登录、建成员/审核人、建工单、普通成员改审核被拒(403)、审核人改为 rejected 后汇总计入(final=5)、feedback/wiki 提交、删除均正常。
- 注意：当前沙箱内 3000 端口曾被旧进程占用，已清理；预览可用 3100（已运行的更新版服务）。
